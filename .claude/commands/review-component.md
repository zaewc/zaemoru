# Review Component

Review a Zaemoru component for design-system quality.

Check:

1. API simplicity
2. Token usage
3. Framework independence
4. Web Component correctness
5. React adapter correctness
6. Accessibility
7. Keyboard interaction
8. Focus state
9. Disabled state
10. Event naming
11. Docs coverage
12. Build/type safety

Output format:

## Verdict

Good / Needs changes / Blocker

## Issues

List concrete issues with file paths.

## Required changes

Give specific patch instructions.

## Optional improvements

Give non-blocking suggestions.
